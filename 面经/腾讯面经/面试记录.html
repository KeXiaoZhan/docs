# 1.为什么es6越来越少用闭包了？

let和const块级作用域可以不用生成闭包函数来保存循环变量

# 2.vue3的composition api比vue2的option api好在哪里，缺点又有什么？

  • 组合式api代码的逻辑相关性更强，选项式api将属性和方法存放到不同的地方，也不方便阅读代码
  • vue2中通过mixin重用逻辑代码，有可能会发生命名冲突和变量来源不明确的问题

# 9.箭头函数和普通函数的区别

# 11.为什么要用vnode

虚拟dom虽然比不上原生js操作dom的速度，但可以降低程序员的心智负担

用JS对象（虚拟dom）模拟DOM节点的好处是，页面的更新可以先全部反映在JS对象(虚拟DOM)上，心智负担小，毕竟操作虚拟 DOM 其实操作的只是一个对象。而操作对象对我们来说比一个一个选中 DOM ，修改 DOM 简单太多了，因为只需要关心数据嘛。

# 13.设计一个登陆注册功能，前端后端，ui，产品都要做什么

前端

前端主要负责与用户交互，呈现用户界面，并将用户输入的数据发送到后端。下面是前端需要完成的任务：

  1. 设计用户界面，包括登陆和注册页面。
  2. 实现用户输入验证，确保用户输入的数据格式正确并符合要求。
  3. 将用户输入的数据发送到后端进行处理。
  4. 处理后端返回的数据，并根据返回的数据在用户界面上做出相应的反馈。
  5. 管理用户的会话状态，记录用户的登录状态，保持用户登录状态的持久性。

后端

后端主要负责处理用户数据，并将结果返回给前端。下面是后端需要完成的任务：

  1. 接收前端发送的请求，解析请求参数，处理请求。
  2. 验证用户输入的数据，确保用户输入的数据合法并符合要求。
  3. 处理用户数据，例如创建用户账户、验证用户身份等。
  4. 返回结果，例如成功或失败的信息以及用户数据。

UI

UI设计师需要负责设计用户界面，确保用户界面易于使用、美观和符合品牌形象。下面是UI设计师需要完成的任务：

  1. 确定用户界面的样式、颜色、字体和排版。
  2. 设计用户交互，例如按钮、文本框、下拉框等。
  3. 与前端合作确定用户界面的交互流程。
  4. 提供用户界面的视觉设计和原型。

产品

产品经理需要负责设计用户流程，确保整个登录注册功能的使用流程顺畅、易于理解，并符合用户需求。下面是产品经理需要完成的任务：

  1. 分析用户需求，确定用户的主要痛点和需求。
  2. 设计用户流程，确保用户可以顺利完成整个登录注册功能的使用。

# 20.vue2中当data里的对象被添加新属性时视图是否会刷新，为什么

# 22.有没有了解过跨域，为什么会出现跨域，如果关闭浏览器的同源限制是否还需要跨域，跨域的方法有哪些

跨域是由于浏览器的同源策略（Same-Origin Policy）引起的。同源策略是浏览器的一种安全策略，限制了一个源的文档或脚本如何与另一个源的资源进行交互，以避免跨域攻击。在浏览器中，同源是指协议、域名和端口号都相同。

如果关闭浏览器的同源限制，则不需要跨域。但这会让浏览器处于非常不安全的状态，因为恶意网站可以通过在用户不知情的情况下访问其他站点的资源来攻击用户的数据和隐私。因此，不建议关闭浏览器的同源限制。

有多种方法可以解决跨域问题，下面是一些常见的方法：

  • JSONP：利用<script>标签可以跨域加载资源的特性，通过回调函数实现跨域数据传输。
  • CORS：服务端设置响应头中的Access-Control-Allow-Origin字段，允许指定的源访问资源。
  • 代理服务器：客户端发送请求到同源的代理服务器，由代理服务器再发送跨域请求并将响应返回给客户端。
  • WebSocket：基于HTTP协议的WebSocket可以建立跨域的双向通信通道。
  • postMessage：跨窗口通信API，允许在不同窗口之间进行安全的跨域数据传输。

# 26.vue-router hash history模式区别以及相关原理

Vue Router 是 Vue.js 官方的路由管理器。Vue Router 为单页面应用提供了强大的路由功能，可以通过 URL 映射到对应的组件。Vue Router 提供两种路由模式：history 和 hash。

在 history 模式下，路由路径中不再需要 # 符号，如下所示：

原理：

# 28.data为什么是函数不是对象

这主要是由于Vue的组件实例是可以复用的。

当我们在一个Vue应用中使用同一个组件多次时，如果每个组件的data属性都是同一个对象的话，那么它们会共享同一个数据对象，一个组件对数据的修改会影响到其他组件。

# 38.webpack的打包流程

  1. 读取配置文件：Webpack 在执行时会先读取配置文件，根据配置文件中的内容来决定如何打包项目。
  2. 解析模块依赖：Webpack 会从入口文件开始，递归地解析所有的模块依赖，构建出一个依赖关系图，该图包含了所有模块之间的依赖关系。

# 41.浏览器的渲染的过程

  1. 解析 HTML 标记并构建 DOM 树：浏览器会将 HTML 标记解析成 DOM 树，即 Document Object Model，DOM 树是由一个个节点构成的树形结构，表示了 HTML 文档的结构和内容。
  2. 解析 CSS 样式并构建 CSSOM 树：浏览器会将 CSS 样式解析成 CSSOM 树，即 CSS Object Model，CSSOM 树是由一个个样式规则构成的树形结构，表示了 HTML 文档的样式信息。
  3. 将 DOM 树和 CSSOM 树结合生成渲染树：渲染树是由 DOM 树和 CSSOM 树结合生成的，它只包含需要显示的节点和样式信息。渲染树的生成过程中，浏览器会根据 CSS 样式计算每个节点的大小、位置等布局信息。
  4. 布局和绘制：浏览器会根据渲染树进行布局和绘制，即将渲染树的节点按照其在视口中的位置进行排布，并将它们绘制到屏幕上。
  5. 页面重绘和回流：当页面中的某些元素发生变化时，浏览器会重新进行渲染，这个过程分为重绘和回流。重绘是指浏览器会重新绘制发生变化的元素，但是不会改变它们的位置和大小。回流是指浏览器会重新计算页面布局，当发生回流时，所有受影响的元素都需要重新计算其位置和大小，然后重新排布和绘制，这个过程比重绘的代价更高，应该尽量避免。

# 49 js中基本类型和引用类型的区别

在JavaScript中，数据类型可以分为以下两类：

  1. 基本数据类型（Primitive Types）：包括数字(Number)、字符串(String)、布尔(Boolean)、空值(null)、未定义(undefined)以及ES6新增的符号(Symbol)。
  2. 引用数据类型（Reference Types）：包括对象(Object)、数组(Array)、函数(Function)等。

区别:

  1. 存储方式：基本类型的值直接存储在栈内存中，而引用类型的值存储在堆内存中，而栈内存中存储的是堆内存地址。
  2. 复制方式：基本类型赋值时是将值复制到新的变量中，而引用类型的赋值时只是复制了引用地址，多个变量指向同一块内存地址，修改其中一个变量的值会影响其他变量。
  3. 比较方式：基本类型比较时是值的比较，而引用类型比较时是引用地址的比较。
  4. 生命周期：基本类型的生命周期只在其所在的作用域内，而引用类型的生命周期可以跨越多个作用域，直到没有任何变量引用它时被垃圾回收器回收。

# 61 为什么你 token 要保存到 localStorage 里面，放 cookie 里面不行吗

那么为什么一般建议将 Token 存储在 localStorage 中呢？

  1. 安全性：将 Token 存储在 localStorage 中可以增加应用程序的安全性。因为 cookie 存储在客户端，所以可以被其他脚本或攻击者轻易地访问和窃取，而 localStorage 只能被同源的 JavaScript 访问，因此比 cookie 更加安全。
  2. 大小限制： cookie 的大小通常被限制在 4KB 左右，而 localStorage 可以存储更多的数据，因此在需要存储大量数据的情况下，localStorage 更加适合。
  3. 防止 CSRF 攻击： 将 Token 存储在 cookie 中容易受到跨站请求伪造（CSRF）攻击的威胁。攻击者可以伪造用户请求，以此来获取该用户的 Token。而将 Token 存储在 localStorage 中可以有效地防止此类攻击，因为攻击者不能访问其他域名的 localStorage。

# 73 Vue虚拟domDiff算法流程

Vue 的虚拟 DOM diff 算法流程如下：

  1. 首先比较新旧虚拟 DOM 的根节点是否相同，如果不同，则直接用新的虚拟 DOM 树替换旧的虚拟 DOM 树。
  2. 如果根节点相同，则开始递归比较它们的子节点。首先比较子节点数量是否相同，如果不同，则直接用新的子节点替换旧的子节点。
  3. 如果子节点数量相同，则依次比较每个子节点。对于同一个位置的子节点，首先比较它们的标签名是否相同，如果不同，则直接用新的子节点替换旧的子节点。
  4. 如果标签名相同，则比较它们的属性、事件、样式等是否相同。如果不同，则更新旧的子节点的属性、事件、样式等，否则不做任何处理。
  5. 最后比较子节点的文本内容是否相同，如果不同，则用新的文本内容替换旧的文本内容。

# 3 （重点）HTTP2相较于HTTP1的改进

HTTP2是HTTP协议的第二个主要版本，相较于HTTP1，有以下改进：

  1. 多路复用：HTTP2使用二进制分帧，可以将同一个连接中的多个请求和响应同时进行，避免了HTTP1中的队头阻塞问题，提高了性能。
  2. 数据流优先级：HTTP2支持对数据流设置优先级，可以优先处理重要的请求和响应，提高了用户体验。
  3. 服务器推送：HTTP2可以在客户端请求之前主动将一些资源推送给客户端，避免了客户端重复请求，提高了性能。
  4. 头部压缩：HTTP2使用HPACK算法对HTTP头部进行压缩，减少了头部大小，提高了性能。
  5. 支持SSL/TLS：HTTP2要求使用加密传输，保证了数据的安全性。
  6. 二进制分帧传输：HTTP2中所有传输的数据都是二进制格式，而HTTP1中则是文本格式，提高了传输效率。 HTTP2的这些改进，使得HTTP2在性能、效率、安全性等方面都优于HTTP1，因此越来越多的网站开始采用HTTP2协议。

# 6 vue computed原理

计算属性的原理如下：

  1. 在组件实例化时，Vue会遍历所有的计算属性，并为每个计算属性创建一个Watcher实例。
  2. 在Watcher实例化时，会执行计算属性的getter函数，并将getter函数中访问的所有响应式数据作为依赖收集起来，并将Watcher实例添加到这些响应式数据的依赖列表中。
  3. 当依赖发生改变时，响应式数据会通知它的所有依赖，依赖会通知它们所管理的所有Watcher实例。
  4. 当Watcher实例收到更新通知时，会重新执行计算属性的getter函数，并将计算结果缓存起来。
  5. 当下次访问计算属性时，如果该计算属性的依赖没有发生改变，则直接返回缓存的计算结果，否则重新计算。 通过以上原理，Vue可以避免不必要的计算，提高计算效率，并确保计算结果的正确性。

可以理解为观察者模式，观察响应式数据是否改变，改变了就更新计算属性

function deepClone3(target,map = new Map()) { 
    //检测数据类型
    if (typeof target !== 'object' || target === null) {
        // target 是 null ，或者不是对象和数组，说明是原始类型,直接返回
        return target;
    } else { 
        //克隆数据前，先进行判断数据之前是否克隆过
        let cache = map.get(target);
        //如果克隆过,直接返回映射中键对应的值,绕开了递归
        if(cache) { 
            return cache;
        }
        //创建一个容器,存储数组或者对象
        const result = Array.isArray(target) ? [] : {};
        //注意:map.set()传入的值是按引用传入,后序result改变也会影响map中保存的状态
        map.set(target,result);
        //遍历target
        for(let key in target) { 
            //检测该属性是否为对象本身的属性(不能拷贝原型对象的属性)
            if(target.hasOwnProperty(key)) { 
                result[key] = deepClone3(target[key],map);
            }
        }
        return result;
    }
}

# 16 浏览器渲染原理

浏览器渲染原理是指浏览器如何将HTML、CSS和JavaScript转换为用户可见的网页的过程。浏览器渲染过程可以分为以下几个步骤：

  1. 构建DOM树：浏览器将HTML文档解析成DOM树，DOM树是由节点（节点类型包括元素节点、文本节点、注释节点等）和它们之间的关系组成的树状结构，表示HTML文档的结构和内容。
  2. 构建CSSOM树：浏览器将CSS样式表解析成CSSOM树，CSSOM树是由样式规则和它们之间的关系组成的树状结构，表示网页的样式信息。
  3. 合并DOM树和CSSOM树：浏览器将DOM树和CSSOM树合并成一棵渲染树（Render Tree），渲染树只包含需要显示的节点和它们的样式信息，不包含不需要显示的节点（例如head、script等节点）。
  4. 布局计算：浏览器根据渲染树中每个节点的样式信息和尺寸计算出每个节点在屏幕上的位置和大小，这个过程称为布局计算。
  5. 绘制渲染树：浏览器根据渲染树中每个节点的位置和大小，将每个节点绘制成屏幕上的像素，这个过程称为绘制渲染树。
  6. 合成图层：浏览器将绘制渲染树的过程分成多个图层，这些图层可以独立绘制和合成，从而提高渲染性能和用户体验。 以上是浏览器渲染原理的基本过程，不同的浏览器可能会有一些细微的差别，但整体流程是相似的。理解浏览器渲染原理可以帮助开发者更好地优化网页性能和用户体验。

# 17 proxy和Object.defineProperty区别

Proxy和Object.defineProperty都是JavaScript提供的用于监听对象属性变化的机制，它们的区别如下：

  1. **功能不同：Object.defineProperty只能监听单个属性的读取和写入操作，而Proxy可以监听对象的多种操作，**包括读取、写入、删除、枚举、函数调用等。
  2. 兼容性不同：Object.defineProperty在ES5中被引入，兼容性较好，但是不能监听数组的变化。Proxy在ES6中被引入，兼容性较差，需要进行兼容性处理，但是可以监听数组的变化。
  3. 性能不同：Object.defineProperty对于单个属性的读取和写入操作性能较好，但是在监听多个属性时，需要为每个属性都定义getter和setter方法，会导致代码的冗余和性能的下降。而Proxy只需要一个handler对象来监听多个属性，可以减少代码量和性能损耗。

# 20 异步编程方法

在 JavaScript 中，异步编程方法有以下几种：

  1. 回调函数：回调函数是一种最基本的异步编程方法。通过将函数作为参数传递给另一个函数，当另一个函数执行完毕后，再调用传入的函数来处理结果。
  2. Promise：Promise 是一种异步编程模型，可以解决回调函数的嵌套问题。Promise 通过链式调用 then 方法，将多个异步操作串行执行。
  3. async/await：async/await 是 ES7 中新增的异步编程方式。async/await 可以将异步操作像同步操作一样进行编写，使异步代码更加易读易写。

  5. 发布/订阅模式（观察者模式）：发布/订阅模式是一种解耦的异步编程方式。通过订阅事件来处理异步操作的结果，可以使代码更加灵活可扩展。
  6. Generator：Generator 是一种异步编程方式，可以通过 yield 和 next 方法实现异步操作的同步化编写。 以上这些异步编程方法各有优缺点，根据实际场景选择合适的方式可以提高代码的可读性和可维护性。

# 22 v-for需要确定key，为什么

Vue 中，每个虚拟 DOM 都需要唯一的 key 值来标识自己，以便在更新视图时能够正确地找到对应的虚拟 DOM 进行更新。如果没有设置 key 值，Vue 会给出警告，并采用默认的 key 值，可能会导致渲染错误或性能降低。 设置 key 值的好处包括：

  2. 避免出现渲染错误：如果没有设置 key 值，可能会导致渲染错误，例如删除数组中的某个元素后，后面的元素会向前移动，但是没有设置 key 值时，Vue 可能会认为这些元素没有变化，从而不会进行重新渲染，导致出现渲染错误。

# 26 echarts的canvas和svg渲染有什么差别

ECharts 是一款基于 Canvas 和 SVG 渲染的开源可视化库，两者渲染方式的差别如下：

  1. 渲染方式不同：Canvas 是基于像素的渲染方式，使用 JavaScript 操作像素点来绘制图形；SVG 是基于矢量的渲染方式，使用 XML 描述图形信息，并通过浏览器渲染引擎将其转化为像素点。
  2. **性能优劣不同：Canvas 的渲染速度比 SVG 快，特别是在大量数据渲染时，Canvas 的性能更优。**但是 Canvas 绘制的图形不支持事件绑定，对于交互性要求高的图表不太适用。而 SVG 的渲染速度相对较慢，但是支持事件绑定，对于交互性要求高的图表更加适用。
  3. 动画效果不同：由于 Canvas 是像素级别的渲染方式，支持像素级别的动画效果。而 SVG 是基于矢量的渲染方式，支持元素级别的动画效果。
  4. 兼容性不同：Canvas 对浏览器的兼容性较好，但是不支持一些低版本浏览器。而 SVG 对浏览器的兼容性较差，需要进行兼容性处理。 综上所述，Canvas 和 SVG 渲染方式各有优劣，并且适用于不同的场景。在选择渲染方式时，需要根据具体需求和场景来进行选择。

# 27 前后端如何协作

前后端协作是一个完整的 Web 应用程序开发过程中非常重要的部分。下面是前后端协作的一些重要方面：

  1. 接口约定 前后端应该确定好接口的请求方式、请求参数、返回数据格式等，以便前端能够正确地调用后端接口。接口约定可以在开发前进行讨论，也可以通过 Swagger 等接口文档工具进行生成和共享。
  2. 接口开发 后端开发人员需要按照接口约定，开发出符合前端需求的接口。在开发过程中，后端人员需要关注接口的性能、安全性等问题。
  3. 接口测试 前端开发人员需要对接口进行测试，确保接口的正确性和稳定性。前端开发人员可以使用 Postman 等工具测试接口，并将测试结果反馈给后端开发人员。
  4. 前端开发 前端开发人员需要根据接口约定，实现页面的交互和数据展示等功能。在开发过程中，前端人员需要考虑页面的性能、用户体验等问题。
  5. 前端测试 开发完成后，前端需要对页面进行测试，确保页面的正确性和稳定性。前端测试可以采用手动测试和自动化测试相结合的方式。
  6. 集成测试 前后端开发完成后，需要进行集成测试，确保前后端的协作正常。集成测试可以在本地进行，也可以在测试服务器上进行。
  7. 上线部署 经过测试后，前后端代码可以部署到生产环境中。在部署过程中，需要注意安全性和稳定性等问题。 总体来说，前后端协作需要开发人员之间的协作和沟通，以便实现高效、高质量的 Web 应用程序。

# 30 如何进行鉴权的

前后端鉴权是保障 Web 应用程序安全的重要手段之一。下面是前后端鉴权的一些常见方式：

  1. 传统 Session 鉴权 在传统的 Web 应用程序中，后端一般使用 Session 来进行用户鉴权。前端在登录后会得到一个 Session ID，后续的请求会将该 Session ID 传递给后端，后端通过验证 Session ID 来进行鉴权。缺点是对于分布式系统或者需要跨域的场景不太适用。
  2. Token 鉴权 Token 鉴权是现代 Web 应用程序中常用的鉴权方式。前端在登录成功后，后端会返回一个 Token，前端将 Token 存储在本地浏览器中。后续的请求会将 Token 附加在请求头中，后端通过验证 Token 来进行鉴权。常见的 Token 包括 JWT 和 OAuth 等。

# 37 从输入url到页面渲染的过程

从输入 URL 到页面渲染的过程主要包括以下步骤：

  1. DNS 解析：浏览器首先会根据输入的 URL 中的域名部分进行 DNS 解析，将域名解析成 IP 地址，以便能够建立 TCP 连接。
  2. 建立 TCP 连接：在浏览器和服务器之间建立 TCP 连接，通过三次握手协议确认连接的可靠性。
  3. 发送 HTTP 请求：浏览器向服务器发送 HTTP 请求，请求中包含请求方法、请求头、请求体等信息。
  4. 服务器响应请求：服务器收到请求后，根据请求信息生成响应报文，并发送给浏览器。
  5. 接收响应内容：浏览器接收到响应报文后，对其进行解析，获取响应头和响应体中的数据。
  6. 解析 HTML：浏览器根据响应体中的 HTML 文档进行解析，构建 DOM 树。
  7. 解析 CSS：浏览器根据 HTML 中的样式信息和外部样式表中的样式信息解析 CSS 样式，并生成 CSSOM 树。
  8. 构建渲染树：浏览器将 DOM 树和 CSSOM 树结合起来，生成渲染树。
  9. 布局计算：浏览器根据渲染树中每个元素的大小、位置等信息进行布局计算，得到每个元素在屏幕中的位置。
  10. 绘制页面：最后，浏览器将渲染树中的每个元素绘制到屏幕上，完成页面的渲染。 以上是浏览器渲染页面的主要过程，其中还包括一些优化技术，例如 DNS 缓存、HTTP 缓存、预解析等，以提高页面加载速度和性能。

# 38 js有哪些数据类型，基本类型和引用类型的区别

JavaScript 有 7 种数据类型，分为基本数据类型和引用数据类型：

  1. 基本数据类型（Primitive Types）：包括 Undefined、Null、Boolean、Number、String 和 Symbol（ES6 新增）这 6 种类型。
  2. 引用数据类型（Object Types）：包括 Object、Array、Function 和 Date 等类型。 基本数据类型和引用数据类型在存储方式和使用方式上有很大的区别：
  3. 存储方式不同：基本数据类型的值直接存储在栈（stack）中，而引用数据类型的值存储在堆（heap）中，栈中存储的实际上是该对象在堆中的引用地址。
  4. 复制变量值的方式不同：在将一个基本数据类型的值赋值给另一个变量时，会将原值复制一份到新的变量的位置。而在将一个引用数据类型的值赋值给另一个变量时，实际上复制的是该对象在堆中的引用地址，因此两个变量最终指向同一个对象。
  5. 传参方式不同：在将一个基本数据类型的值作为参数传递给函数时，会将该值复制一份到函数的参数中，因此在函数内部修改该参数的值不会影响到原来的变量的值。而在将一个引用数据类型的值作为参数传递给函数时，实际上传递的是该对象在堆中的引用地址，因此在函数内部修改该参数的值会影响到原来的变量的值。 总之，基本数据类型是按值存储的，而引用数据类型是按引用存储的。基本数据类型具有不可变性（immutability），而引用数据类型可以被修改。在使用中需要注意它们的差异。

# 39 简述下有哪些排序算法，并简述原理

常见的排序算法主要包括以下几种：

  1. 冒泡排序：从左往右依次比较相邻的两个数，如果前面的数大于后面的数就交换它们的位置，一轮比较下来，最大的数就会沉到最底下。时间复杂度为 O(n^2)。
  2. 选择排序：每次找到最小的数放到未排序的数列的最前面，直到排序完成。时间复杂度为 O(n^2)。
  3. 插入排序：将未排序的数列中的每个数插入到已排序数列的合适位置，时间复杂度为 O(n^2)。
  4. 快速排序：首先从数列中取出一个数作为基准数，然后将比基准数小的数放在基准数的左边，比基准数大的数放在基准数的右边，然后再递归地对左右两个子序列进行快速排序，直到整个序列有序。时间复杂度为 O(nlogn)。
  5. 归并排序：将数列分成两个子序列，分别对两个子序列进行排序，然后将排序好的两个子序列合并成一个有序的序列。时间复杂度为 O(nlogn)。
  6. 堆排序：将待排序数列构建成一个大根堆或小根堆，然后依次将堆顶的数取出来，放到有序数列的末尾，重复这个过程，直到整个数列有序。时间复杂度为 O(nlogn)。

# 40.讲一下tcp协议

TCP（Transmission Control Protocol，传输控制协议）是一种面向连接的、可靠的、基于字节流的传输层协议，适用于对数据可靠性要求较高的场景，但由于它需要进行“三次握手”等额外的操作，因此相比于 UDP 协议来说，TCP 协议的性能稍微有些低下。

TCP 协议提供了以下几种特性：

  1. 可靠传输：TCP 采用“三次握手”建立连接，保证了数据的可靠传输，因为只有当双方确认建立连接后，数据才会被传输。
  2. 流量控制：TCP 采用“滑动窗口”机制进行流量控制，以防止发送方发送过多数据导致接收方无法处理。
  3. 拥塞控制：TCP 采用“拥塞窗口”机制进行拥塞控制，以防止网络过载。
  4. 面向字节流：TCP 不会记录消息边界，而是将传输的数据视为一连串的无结构字节流。
  5. 双向通信：TCP 支持双向通信，双方都可以发送和接收数据。

# 41.讲一下https协议

HTTPS（Hyper Text Transfer Protocol Secure，安全超文本传输协议）是一种安全的 HTTP 协议，主要解决了 HTTP 协议中存在的安全问题。HTTPS 协议通过使用 SSL/TLS 加密协议来保证数据的传输过程中不被篡改或窃取，从而保证了数据的安全性。

HTTPS 协议相比于 HTTP 协议具有以下优点：

  1. 数据加密：HTTPS 协议使用 SSL/TLS 加密协议对数据进行加密，保证了数据传输的安全性。
  2. 数据完整性：HTTPS 协议使用数字证书验证服务端身份，并使用消息摘要算法保证数据的完整性，从而避免了数据被篡改的风险。
  3. 安全性高：HTTPS 协议使用了多种加密算法和安全协议，可以有效地防止黑客的攻击和窃取。

不过，HTTPS 协议也存在一些缺点，比如会增加服务器的负担和增加数据传输的延迟等。

# 42.讲一下web socket协议

WebSocket 是一种网络通信协议，可以在浏览器和服务器之间建立双向通信，允许服务器主动向客户端推送数据。

WebSocket 协议的实现过程如下：

  1. 客户端向服务端发送一个 HTTP 请求，该请求包含一个 Upgrade 头部，告诉服务器要升级协议到 WebSocket。
  2. 服务器收到客户端的请求之后，会发送一个响应，告诉客户端协议已经升级到 WebSocket。
  3. 客户端和服务器之间建立一个双向连接，可以实现实时通信。

WebSocket 协议相比于 HTTP 协议具有以下优点：

  1. 实时通信：WebSocket 协议可以实现实时通信，不需要像 HTTP 协议一样频繁地建立和关闭连接，节省了网络资源和服务器资源。
  2. 双向通信：WebSocket 协议的通信过程是双向的，可以实现客户端和服务器之间的双向通信，允许服务器主动向客户端推送数据。
  3. 简单易用：WebSocket 协议的 API 很简单，容易学习和使用。

# 45.mongodb和mysql数据库的区别

MongoDB 和 MySQL 是两种不同的数据库系统，它们有一些重要的区别：

  1. 数据模型：MongoDB 是一个文档型数据库，每个文档可以包含多个字段和子文档。而 MySQL 是一个关系型数据库，数据以表格形式存储，每个表格包含多个行和列。
  2. 事务处理：MongoDB 在某些版本中支持事务处理，但不如 MySQL 完善。MySQL 对事务处理的支持非常强大，可以实现 ACID 特性的事务处理。
  3. 索引：MongoDB 支持内置索引和自定义索引，并且支持全文搜索等高级索引技术。而 MySQL 的索引通常是 B-Tree 索引，没有 MongoDB 灵活。
  4. 存储方式：MongoDB 使用 BSON 格式（一种二进制 JSON）进行数据存储，而 MySQL 使用行存储和列存储两种方式。
  5. 扩展性：MongoDB 在水平扩展方面非常强大，可以通过分片技术进行无缝扩展。MySQL 在垂直扩展方面更为强大，可以通过增加硬件资源来提高性能。

总的来说，MongoDB 更适合存储非结构化和半结构化数据，而 MySQL 更适合存储结构化数据。在实际应用中，需要根据具体需求来选择合适的数据库系统。

# 55 tcp四次挥手干了什么事，为什么是四次。如果某次确认报文丢失，会怎么处理?然后问了一些细节，有点忘了

TCP 四次挥手是指 TCP 连接断开时，终止方和发起方共同完成的四个步骤，具体如下：

  1. 终止方发送 FIN 报文，表示要关闭连接，进入 FIN_WAIT_1 状态。
  2. 发起方收到 FIN 报文，发送 ACK 报文作为响应，进入 CLOSE_WAIT 状态。
  3. 发起方发送 FIN 报文，表示自己也要关闭连接，进入 LAST_ACK 状态。
  4. 终止方收到 FIN 报文，发送 ACK 报文作为响应，进入 TIME_WAIT 状态，等待 2MSL 后进入 CLOSED 状态，
  5. 而发起方收到 ACK 报文后，进入 CLOSED 状态。

TCP 四次挥手之所以是四次，是因为在结束连接时需要双方各发送一次 FIN 报文和收到对方的 ACK 报文，因此需要四次通信来完成连接的断开。 如果某次确认报文丢失，可能会导致连接无法正常关闭。在这种情况下，发起方会一直等待终止方发送 ACK 报文，而终止方因为没有收到发起方的 FIN 报文的确认，会一直等待 FIN 报文的到来，从而导致连接处于半关闭状态，无法正常关闭。为了解决这个问题，TCP 采用了超时重传机制，如果发起方在一定时间内没有收到终止方的 ACK 报文，就会重新发送 FIN 报文，直到收到 ACK 报文为止，从而保证连接能够正常关闭。

# 57 ca证书是什么，讲讲整个传递过程

CA（Certificate Authority，证书颁发机构）证书是一种数字证书，用于验证网站或服务器的身份，以确保网络通信的安全性。CA 证书包括网站的公钥、证书过期时间、证书颁发机构的信息等，可以通过浏览器或操作系统内置的信任列表来验证证书的合法性。 整个 CA 证书传递过程一般如下：

  1. 申请证书：网站管理员向 CA 提交证书申请，包括网站的公钥、域名和相关资料。
  2. 颁发证书：CA 对申请进行审核，并为网站颁发数字证书，包括证书的公钥、网站信息和颁发机构信息等。

  4. 客户端验证：当用户访问网站时，网站会将证书发送给客户端，客户端通过内置的信任列表来验证证书的合法性，如果证书合法，则使用证书中的公钥进行加密通信，保证通信的安全性。 需要注意的是，如果证书被篡改或伪造，会导致通信的不安全，因此在使用证书时需要注意保障证书的安全性，如定期更换证书、使用高强度的证书加密算法等。同时，客户端和服务器端也需要保持最新的系统和应用程序版本，以避免已知的安全漏洞。

# 62 vue3响应式原理

Vue 3 使用了 Proxy 对象来实现响应式原理。下面是大致的实现流程：

  1. 在组件初始化时，对组件的 data 对象进行递归遍历，使用 Object.defineProperty 方法将每个属性转换为 getter 和 setter 方法，并且将它们存储在一个依赖收集器中。
  2. 当组件中的 data 对象发生变化时，setter 方法会被触发，更新对应属性的值，并且通知依赖收集器，依赖收集器会遍历存储的所有组件中的依赖，并且触发它们的更新函数。
  3. Vue 3 使用 Proxy 对象来代替 Object.defineProperty 方法，这样可以直接监听整个对象的变化，而不需要遍历每个属性。同时，由于 Proxy 对象可以监听数组的变化，因此 Vue 3 不需要像 Vue 2 一样使用 hack 方法来监听数组的变化。
  4. Vue 3 还引入了 WeakMap 对象来存储依赖关系，这样可以避免内存泄漏的问题，因为 WeakMap 对象中存储的对象会在没有任何引用时被自动回收。 总的来说，Vue 3 的响应式原理和 Vue 2 相比，使用了更加高效的实现方式，并且可以直接监听整个对象的变化，避免了遍历属性的性能问题。

# 72 项目权限管理怎么做的

前端项目的权限控制通常需要涉及以下几个方面：

  1. 用户登录验证：在用户登录之前需要对其身份进行验证，以确保其具有访问该应用程序的权限。可以使用用户名/密码，OAuth，OpenID Connect等进行验证。
  2. 权限管理：确定哪些用户可以访问应用程序的哪些部分。可以将用户分为不同的角色（例如管理员、普通用户、游客等），然后根据角色授予不同的权限。
  3. 路由拦截：当用户访问某个需要授权才能访问的页面时，需要对其进行拦截，并根据用户权限的不同，决定是否允许用户访问。
  4. 数据访问控制：当用户访问某些数据时，需要对其进行访问控制，以确保用户只能访问其具有权限的数据。

权限管理如何实现？

使用路由守卫（router guards）来控制用户访问不同的页面，根据用户的角色或权限来决定是否允许进入某个路由。

使用自定义指令（custom directives）来控制用户操作不同的元素，根据用户的角色或权限来决定是否显示或禁用某个按钮或链接。

使用动态组件（dynamic components）来控制用户查看不同的内容，根据用户的角色或权限来决定渲染哪个组件。

**使用路由守卫实现页面级别的权限控制：**在定义路由时，可以给每个路由添加一个meta属性，用来存储该路由所需的角色或权限。然后，在全局前置守卫（beforeEach）中，可以获取用户的角色或权限，并与目标路由的meta属性进行比较，如果匹配则放行，否则跳转到错误页面或登录页面。

使用自定义指令实现元素级别的权限控制：在创建Vue实例时，可以注册一个全局自定义指令（v-permission），用来绑定用户的角色或权限。然后，在模板中，可以给需要控制的元素添加该指令，并传入一个参数，表示该元素所需的角色或权限。在指令的钩子函数中，可以获取用户和元素的角色或权限，并进行比较，如果匹配则显示元素，否则隐藏或禁用元素。

# 74 prisma和mongodb有什么区别

  1. 数据库类型 Prisma是一种ORM（对象关系映射）框架，支持多种关系型数据库，如MySQL、PostgreSQL和SQLite等。而MongoDB则是一种文档型数据库。
  2. 数据模型 Prisma使用的是基于表的数据模型，每个表对应一个数据实体，每个实体包含多个属性。表之间可以建立关系，并且支持多种关系类型。而MongoDB使用的是基于文档的数据模型，每个文档对应一个数据实体，每个实体包含多个字段。文档之间也可以建立关系，但是只支持一种关系类型。
  3. 查询语言 Prisma使用的是Prisma Client，它是一个类型安全的数据库客户端，使用类似于GraphQL的语言来查询和修改数据。它提供了丰富的操作符和函数，可以方便地进行复杂的查询和变更操作。而MongoDB使用的是MongoDB Query Language（MQL），它是一种基于JSON的查询语言，可以进行类似于SQL的查询和更新操作。
  4. 扩展性 Prisma可以与多种数据库配合使用，支持水平和垂直扩展，可以满足大规模应用的需求。而MongoDB天生就支持分布式架构，可以通过分片和副本集来实现高可用和扩展性。 综上所述，Prisma和MongoDB都有各自的优势和适用场景。如果需要使用关系型数据库，并且需要进行复杂的查询和变更操作，可以选择使用Prisma。如果需要使用文档型数据库，并且需要支持分布式架构，可以选择使用MongoDB。

# 2.MVVM和MVC的特点和优缺点

MVVM (Model-View-ViewModel)

特点

  1. 模型(Model)：表示应用程序中用于处理数据逻辑的部分。
  2. 视图(View)：表示应用程序中处理数据显示的部分。
  3. 视图模型(ViewModel)：作为视图与模型之间的中介，处理用户交互和显示逻辑。

优点

数据驱动视图，程序员只需要关注数据，不需要去关注dom了

  1. 分离了视图和业务逻辑，使得代码更加清晰，易于维护。
  2. 视图模型能够提供更好的单元测试支持。
  3. 可以使用数据绑定技术（双向绑定），实现视图和视图模型的自动同步。

缺点

数据驱动视图，操作虚拟dom后再去真实dom，效率没有直接操作真实dom快，（不过可以减少重流重绘次数）

MVC (Model-View-Controller)

特点

  1. 模型(Model)：表示应用程序中用于处理数据逻辑的部分。
  2. 视图(View)：表示应用程序中处理数据显示的部分。
  3. 控制器(Controller)：表示应用程序中处理用户交互的部分。

优点

  1. 代码组织清晰，易于维护。
  2. 模块化分离，能够实现视图与模型的独立开发。
  3. 可以支持多种用户界面实现方式。

缺点

依赖关系复杂，如果更改了html信息，js可能也要大量更改，程序员需要花大量精力去关注dom，获取哪个dom，然后再更改dom内值，频繁操作dom后也会增加回流重绘的次数

  1. 依赖关系复杂，容易造成代码混乱。
  2. 界面与控制器的紧耦合导致难以进行单元测试。
  3. 不够灵活，难以应对复杂业务场景。

# 对数据分析了解吗？认为数据分析最重要的一点是什么，有什么数据分析工具和方法

数据分析最重要的一点是理解数据，并从中提取有用信息，以便做出决策或预测未来趋势。

数据分析需要使用各种工具和方法，以下是其中一些常见的：

  1. Excel：Excel是一种最基本的数据分析工具，它可以用于处理和分析数据、建立图表和模型。

  2. SQL：SQL是一种结构化查询语言，用于管理和处理数据库中的数据。

  3. Python/R：Python和R是两种常用的编程语言，在数据分析中应用广泛。

  4. Tableau/QlikView：这些工具专门用于数据可视化和探索，可以轻松创建交互式图表和仪表板。

  5. 统计学方法：例如回归分析、聚类分析、决策树、人工神经网络等，用于解释数据、发现关系和预测未来走势。

    总的来说，数据分析需要结合不同的工具和方法，以便从数据中获取更深入的洞察，并提供对业务决策的支持。
